# PPT 디자인 규칙: "프로"의 기준

이 문서는 PPT 생성 시 "전문 디자이너 수준의 품질"을 달성하기 위한 경험적 규칙과 원칙을 정의합니다.

## 1. 콘텐츠-레이아웃 매핑 규칙 ("무엇을 어디에 담을까?")

### 📅 일정 및 타임라인 (Schedule & Timeline)

| 콘텐츠 유형          | 최적 레이아웃                         | 사용 시기                                                  |
| :------------------- | :------------------------------------ | :--------------------------------------------------------- |
| **전체 마스터 일정** | **간트 차트 (Gantt Chart)**           | 기간, 단계, 겹치는 일정 등을 한눈에 보여줄 때              |
| **상위 단계 흐름**   | **쉐브론 프로세스 (Chevron Process)** | 날짜보다는 순차적 진행(1단계 → 2단계)이 중요할 때          |
| **주요 행사 / 출시** | **로드맵 / 저니맵 (Roadmap)**         | 킥오프, 오픈 등 특정 "시점(Point-in-Time)" 위주의 마일스톤 |
| **역사 / 연혁**      | **수직 타임라인 (Vertical Timeline)** | 연대기 순의 역사나 시간 흐름을 리스트 형태로 보여줄 때     |

### 👥 조직 및 인력 (Organization & People)

| 콘텐츠 유형         | 최적 레이아웃                          | 사용 시기                                              |
| :------------------ | :------------------------------------- | :----------------------------------------------------- |
| **보고 체계**       | **계층형 조직도 (Hierarchy Tree)**     | 명확한 "보고 라인(Who reports to whom)"이 중요할 때    |
| **업무 분장 (R&R)** | **프로필 카드 그리드 (Profile Cards)** | 각 인원의 역할과 책임을 상세히 기술할 때 (헤더로 구분) |
| **인력 투입 현황**  | **분할 뷰 (Split View: Chart + List)** | 좌측: 조직 구조 / 우측: 요약 통계(총 M/M, 인원 수)     |
| **파트너 / 생태계** | **로고 그리드** 또는 **방사형 맵**     | 중심 기업과 파트너사 간의 관계를 보여줄 때             |

### 📊 데이터 및 지표 (Data & Metrics)

| 콘텐츠 유형              | 최적 레이아웃                       | 사용 시기                                             |
| :----------------------- | :---------------------------------- | :---------------------------------------------------- |
| **핵심 성과 지표 (KPI)** | **빅 넘버 카드 (Big Number Cards)** | "**30%** 비용 절감" 처럼 하나의 숫자가 매우 중요할 때 |
| **진척률 / 달성률**      | **도넛 차트 (Donut Chart)**         | "75% 완료" 등 전체 대비 부분을 보여줄 때              |
| **비교**                 | **막대 차트 (Bar / Column Chart)**  | 항목 간 값의 크기를 비교할 때 (전 vs 후)              |
| **추세 / 흐름**          | **선 차트 (Line / Area Chart)**     | 시간 경과에 따른 변화 추이를 보여줄 때                |

### 🔄 프로세스 및 논리 (Process & Logic)

| 콘텐츠 유형        | 최적 레이아웃                        | 사용 시기                                         |
| :----------------- | :----------------------------------- | :------------------------------------------------ |
| **반복 프로세스**  | **순환 다이어그램 (Cycle Diagram)**  | 피드백 루프, 애자일 스프린트, PDCA 사이클         |
| **구성 요소 구조** | **방사형 / 허브 앤 스포크 (Radial)** | 중앙 코어와 연결된 요소들 (예: 플랫폼 아키텍처)   |
| **필터링 / 영업**  | **깔때기형 (Funnel Diagram)**        | 100개 리드 → 10개 가망 → 1개 수주 (좁아지는 과정) |
| **단계별 절차**    | **번호 리스트 / 스텝 박스**          | 정확한 지침이나 매뉴얼 성격의 순서                |

## 2. 아이콘 및 비주얼 언어 규칙

### 아이콘 사용 표준 (Icon Usage Standards)

- **일관성 (Consistency)**: **라인(Line)** 스타일과 **솔리드(Solid/Filled)** 스타일을 섞어 쓰지 마십시오.
  - _Pro Tip_: 현대적이고 기술적인 느낌에는 **라인 아이콘**이 좋습니다.
  - _Pro Tip_: 버튼이나 강조가 필요한 곳에는 **솔리드 아이콘**을 쓰세요.
- **문맥 매핑 (Contextual Mapping)**:
  - 🚀 **로켓**: 착수, 시작, 스타트업, 빠름
  - 🛡️ **방패**: 보안, 품질, 리스크 관리, 안전
  - ⚙️ **기어(톱니바퀴)**: 설정, 엔진, 백엔드, 기술
  - 👥 **사용자/그룹**: 팀, HR, 고객, 협업
  - 🎯 **타겟/과녁**: 목표, 미션, 지향점
  - 💡 **전구**: 아이디어, 혁신, 전략

### 타이포그래피 표준 (Typography Standards)

- **계층 구조 (Hierarchy)**: 제목 (Bold 24pt+) > 부제목 (Regular 14-16pt) > 본문 (Light/Regular 10-12pt).
- **가독성 (Readability)**: 폰트는 **최대 2개**까지만 사용하세요.
  - _안전한 선택_: Arial / Roboto / Pretendard (산세리프 계열)
  - _강조_: 숫자와 키워드에는 **Bold**를 사용하고, 설명에는 *Light*나 Regular를 사용하여 대비를 줍니다.

## 3. 구도 및 여백 ("황금율")

- **3분할 법칙 (Rule of Thirds)**: 모든 것을 정중앙에 두지 마십시오. 시선을 그는 핵심 비주얼은 1/3 지점에 배치하는 것이 안정적입니다.
- **여백의 미 (Whitespace)**: 꽉 찬 슬라이드는 아마추어 같습니다. 모든 가장자리에 최소 **40-60px**의 여백을 확보하세요.
- **정렬 (Alignment)**: 모든 요소는 **반드시** 다른 요소와 정렬선이 맞아야 합니다 (좌측 정렬 권장).
- **그룹핑 (Grouping)**: 관련된 항목끼리는 가깝게, 관련 없는 항목과는 멀게 배치하여 시각적 그룹을 만드세요 (근접성의 원리).

## 4. 고급 "프로" 기법 (Advanced Techniques)

- **고스트 넘버 (Ghost Numbers)**: 리스트 배경에 아주 크고 연한 숫자(투명도 10%)를 깔아주면, 세련된 구조감을 줍니다.
- **글래스모피즘 (Glassmorphism)**: 이미지 위에 텍스트를 올릴 때, 반투명한 흰색 배경에 블러(Blur) 효과를 주어 세련미와 가독성을 동시에 잡으세요.
- **브릿지 & 커넥터 (Connector)**: 프로세스 단계 사이에는 반드시 시각적 연결 고리(선, 화살표)를 넣어 "흐름"이 끊기지 않도록 하세요.
