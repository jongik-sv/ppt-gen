# PPT 디자인 규칙: "프로"의 기준

이 문서는 PPT 생성 시 "전문 디자이너 수준의 품질"을 달성하기 위한 경험적 규칙과 원칙을 정의합니다.

## 1. 콘텐츠-레이아웃 매핑 규칙 ("무엇을 어디에 담을까?")

### 🎬 구조 슬라이드 (Structural Slides)

#### 표지 슬라이드 (Cover / Title Slide)

| 요소 | 권장 사항 | 비고 |
| :--- | :-------- | :--- |
| **제목** | 60pt 이상, Bold, 최상단 또는 중앙 | 슬라이드의 주인공 |
| **부제목** | 제목의 50-60% 크기, Regular | 날짜, 이벤트명, 한 줄 설명 |
| **로고** | 코너 배치 (우상단/우하단), 작고 절제된 크기 | 브랜딩 유지 |
| **배경** | 단색, 그라디언트, 또는 여백 많은 이미지 | 텍스트 가독성 확보 |
| **발표자 정보** | 하단 또는 좌하단, 작은 폰트 | 이름, 직책, 날짜 |

**표지 레이아웃 유형:**

| 레이아웃 유형 | Key | 설명 | 사용 시기 |
| :------------ | :-- | :--- | :-------- |
| **중앙 정렬형** | `cover-center` | 제목 + 부제목 중앙 배치 | 공식적, 심플한 발표 |
| **좌측 정렬형** | `cover-left` | 텍스트 좌측, 우측 여백/이미지 | 현대적, 비즈니스 |
| **이미지 오버레이형** | `cover-overlay` | 풀스크린 이미지 + 반투명 오버레이 + 텍스트 | 임팩트, 크리에이티브 |
| **분할형 (Split)** | `cover-split` | 좌측 텍스트 / 우측 이미지 (또는 반대) | 제품, 서비스 소개 |
| **미니멀형** | `cover-minimal` | 로고 + 제목만 | 브랜드 중심, 티저 |

#### 목차 슬라이드 (Table of Contents / Agenda)

| 요소 | 권장 사항 | 비고 |
| :--- | :-------- | :--- |
| **항목 수** | 3-7개 권장 (최대 9개) | 인지 부하 최소화 |
| **넘버링** | 숫자 또는 아이콘으로 시각적 구분 | 01, 02, 03 또는 ●, ●, ● |
| **시간 배분** | 선택적으로 각 항목별 시간 표기 | 미팅/워크숍용 |
| **현재 위치 표시** | 진행 중인 섹션 하이라이트 | 프로그레스 바 또는 색상 강조 |

**목차 레이아웃 유형:**

| 레이아웃 유형 | Key | 설명 | 사용 시기 |
| :------------ | :-- | :--- | :-------- |
| **수직 리스트형** | `toc-vertical-list` | 번호 + 항목명 세로 나열 | 표준, 간결한 구조 |
| **타임라인형** | `toc-timeline` | 수평 흐름으로 섹션 표시 | 순차적 스토리 |
| **그리드/카드형** | `toc-grid` | 2x2, 2x3 카드로 섹션 배치 | 비선형, 모듈형 |
| **아이콘 강조형** | `toc-icon` | 각 섹션별 대표 아이콘 + 제목 | 시각적 구분 강화 |
| **프로그레스 바형** | `toc-progress` | 상단에 진행 바 + 현재 위치 강조 | 긴 프레젠테이션 |

#### 섹션 구분 슬라이드 (Section Divider / Chapter Break)

| 요소 | 권장 사항 | 비고 |
| :--- | :-------- | :--- |
| **섹션 번호** | 큰 숫자 (100pt+), 연한 색 또는 Bold | "01", "02" 고스트 넘버 스타일 |
| **섹션 제목** | 40-60pt, 중앙 또는 좌측 정렬 | 명확하고 간결하게 |
| **부제목/설명** | 선택적, 1줄 이내 | 섹션 내용 요약 |
| **배경** | 브랜드 컬러, 그라디언트, 패턴 | 콘텐츠 슬라이드와 차별화 |

**섹션 구분 레이아웃 유형:**

| 레이아웃 유형 | Key | 설명 | 사용 시기 |
| :------------ | :-- | :--- | :-------- |
| **풀컬러 배경형** | `divider-fullcolor` | 브랜드 색상 배경 + 흰색 텍스트 | 강한 시각적 전환 |
| **고스트 넘버형** | `divider-ghost-number` | 초대형 연한 숫자 + 섹션 제목 | 세련된 구조감 |
| **이미지 배경형** | `divider-image` | 섹션 관련 이미지 + 오버레이 + 텍스트 | 감성적, 스토리텔링 |
| **미니멀 라인형** | `divider-minimal` | 수평선/수직선 + 섹션 제목만 | 클린, 모던 |
| **아이콘 중심형** | `divider-icon` | 큰 아이콘 + 섹션 제목 | 주제 시각화 |

#### 마무리 슬라이드 (Closing / Thank You)

| 레이아웃 유형 | Key | 설명 | 사용 시기 |
| :------------ | :-- | :--- | :-------- |
| **Thank You형** | `closing-thankyou` | 감사 메시지 + 연락처/QR코드 | 일반 발표 마무리 |

| **Q&A형** | `closing-qna` | "Questions?" + 연락처 | 질의응답 유도 |
| **요약형** | `closing-summary` | 핵심 메시지 3가지 재강조 | 기억에 남기기 |
| **연락처형** | `closing-contact` | 이름, 이메일, 전화, SNS, QR코드 | 네트워킹, 후속 연락 |

### 📅 일정 및 타임라인 (Schedule & Timeline)

#### 기본 타임라인 (Basic Timeline)

| 콘텐츠 유형          | 최적 레이아웃                         | Key                  | 사용 시기                                                  |
| :------------------- | :------------------------------------ | :------------------- | :--------------------------------------------------------- |
| **전체 마스터 일정** | **간트 차트 (Gantt Chart)**           | `gantt-chart`        | 기간, 단계, 겹치는 일정 등을 한눈에 보여줄 때              |
| **상위 단계 흐름**   | **쉐브론 프로세스 (Chevron Process)** | `chevron-process`    | 날짜보다는 순차적 진행(1단계 → 2단계)이 중요할 때          |
| **주요 행사 / 출시** | **로드맵 / 저니맵 (Roadmap)**         | `roadmap`            | 킥오프, 오픈 등 특정 "시점(Point-in-Time)" 위주의 마일스톤 |
| **역사 / 연혁**      | **수직 타임라인 (Vertical Timeline)** | `vertical-timeline`  | 연대기 순의 역사나 시간 흐름을 리스트 형태로 보여줄 때     |
| **핵심 이정표**      | **마일스톤 차트 (Milestone Chart)**   | `milestone-chart`    | 다이아몬드 마커로 주요 체크포인트만 강조 (경영진 보고용)   |
| **분기/연간 계획**   | **캘린더 뷰 (Calendar View)**         | `calendar-view`      | 월별/분기별 주요 일정 한눈에 파악                          |

#### 애자일/스프린트 (Agile & Sprint)

### 👥 조직 및 인력 (Organization & People)

| 콘텐츠 유형             | 최적 레이아웃                          | Key                | 사용 시기                                              |
| :---------------------- | :------------------------------------- | :----------------- | :----------------------------------------------------- |
| **보고 체계**           | **계층형 조직도 (Hierarchy Tree)**     | `org-hierarchy`    | 명확한 "보고 라인(Who reports to whom)"이 중요할 때    |
| **업무 분장 (R&R)**     | **프로필 카드 그리드 (Profile Cards)** | `profile-cards`    | 각 인원의 역할과 책임을 상세히 기술할 때 (헤더로 구분) |
| **인력 투입 현황**      | **분할 뷰 (Split View: Chart + List)** | `split-view`       | 좌측: 조직 구조 / 우측: 요약 통계(총 M/M, 인원 수)     |
| **파트너 / 생태계**     | **로고 그리드** 또는 **방사형 맵**     | `partner-grid`     | 중심 기업과 파트너사 간의 관계를 보여줄 때             |
| **책임 할당 (RACI)**    | **RACI 매트릭스 (테이블)**             | `raci-matrix`      | 업무별 Responsible/Accountable/Consulted/Informed 지정 |
| **이해관계자 분석**     | **Stakeholder Map (2x2 또는 동심원)**  | `stakeholder-map`  | 영향력 vs 관심도 기준 이해관계자 분류                  |
| **역량 매트릭스**       | **Skills Matrix (히트맵 테이블)**      | `skills-matrix`    | 팀원별 스킬 레벨 시각화 (초급/중급/고급)               |
| **팀 구성**             | **Team Canvas (9블록)**                | `team-canvas`      | 팀 목표, 역할, 가치, 규칙 등 팀 빌딩 워크숍            |

### 📊 데이터 및 지표 (Data & Metrics)

#### 기본 차트 유형 (Basic Charts)

| :----------------------- | :---------------------------------- | :------------------ | :---------------------------------------------------- |

| **증감 분석**            | **워터폴 차트 (Waterfall Chart)**   | `waterfall-chart`   | 초기값 → 증가/감소 요인 → 최종값 흐름                 |

### 🔄 프로세스 및 논리 (Process & Logic)

#### 기본 프로세스 다이어그램 (Basic Process Diagrams)

| 콘텐츠 유형        | 최적 레이아웃                        | Key                  | 사용 시기                                         |
| :----------------- | :----------------------------------- | :------------------- | :------------------------------------------------ |
| **반복 프로세스**  | **순환 다이어그램 (Cycle Diagram)**  | `cycle-diagram`      | 피드백 루프, 애자일 스프린트, PDCA 사이클         |
| **구성 요소 구조** | **방사형 / 허브 앤 스포크 (Radial)** | `hub-spoke`          | 중앙 코어와 연결된 요소들 (예: 플랫폼 아키텍처)   |
| **필터링 / 영업**  | **깔때기형 (Funnel Diagram)**        | `funnel-diagram`     | 100개 리드 → 10개 가망 → 1개 수주 (좁아지는 과정) |
| **단계별 절차**    | **번호 리스트 / 스텝 박스**          | `step-list`          | 정확한 지침이나 매뉴얼 성격의 순서                |
| **선형 프로세스**  | **쉐브론 / 화살표 체인**             | `arrow-chain`        | 순차적 단계 (1단계 → 2단계 → 3단계)               |
| **병렬 프로세스**  | **분기형 플로우차트**                | `branch-flowchart`   | 동시에 진행되는 여러 작업 흐름                    |

#### 고급 워크플로우 (Advanced Workflow)

## 2. 아이콘 및 비주얼 언어 규칙

### 아이콘 사용 표준 (Icon Usage Standards)

- **일관성 (Consistency)**: **라인(Line)** 스타일과 **솔리드(Solid/Filled)** 스타일을 섞어 쓰지 마십시오.
  - _Pro Tip_: 현대적이고 기술적인 느낌에는 **라인 아이콘**이 좋습니다.
  - _Pro Tip_: 버튼이나 강조가 필요한 곳에는 **솔리드 아이콘**을 쓰세요.
- **문맥 매핑 (Contextual Mapping)**:
  - 🚀 **로켓**: 착수, 시작, 스타트업, 빠름
  - 🛡️ **방패**: 보안, 품질, 리스크 관리, 안전
  - ⚙️ **기어(톱니바퀴)**: 설정, 엔진, 백엔드, 기술
  - 👥 **사용자/그룹**: 팀, HR, 고객, 협업
  - 🎯 **타겟/과녁**: 목표, 미션, 지향점
  - 💡 **전구**: 아이디어, 혁신, 전략

### 타이포그래피 표준 (Typography Standards)

- **계층 구조 (Hierarchy)**: 제목 (Bold 24pt+) > 부제목 (Regular 14-16pt) > 본문 (Light/Regular 10-12pt).
- **가독성 (Readability)**: 폰트는 **최대 2개**까지만 사용하세요.
  - _안전한 선택_: Arial / Roboto / Pretendard (산세리프 계열)
  - _강조_: 숫자와 키워드에는 **Bold**를 사용하고, 설명에는 *Light*나 Regular를 사용하여 대비를 줍니다.

## 3. 구도 및 여백 ("황금율")

- **3분할 법칙 (Rule of Thirds)**: 모든 것을 정중앙에 두지 마십시오. 시선을 그는 핵심 비주얼은 1/3 지점에 배치하는 것이 안정적입니다.
- **여백의 미 (Whitespace)**: 꽉 찬 슬라이드는 아마추어 같습니다. 모든 가장자리에 최소 **40-60px**의 여백을 확보하세요. 그렇다고 30% 이상 여백이 많으면 슬라이드가 비어보일 수 있습니다.
- **정렬 (Alignment)**: 모든 요소는 **반드시** 다른 요소와 정렬선이 맞아야 합니다 (좌측 정렬 권장).
- **그룹핑 (Grouping)**: 관련된 항목끼리는 가깝게, 관련 없는 항목과는 멀게 배치하여 시각적 그룹을 만드세요 (근접성의 원리).

## 4. 고급 "프로" 기법 (Advanced Techniques)

- **고스트 넘버 (Ghost Numbers)**: 리스트 배경에 아주 크고 연한 숫자(투명도 10%)를 깔아주면, 세련된 구조감을 줍니다.
- **글래스모피즘 (Glassmorphism)**: 이미지 위에 텍스트를 올릴 때, 반투명한 흰색 배경에 블러(Blur) 효과를 주어 세련미와 가독성을 동시에 잡으세요.
- **브릿지 & 커넥터 (Connector)**: 프로세스 단계 사이에는 반드시 시각적 연결 고리(선, 화살표)를 넣어 "흐름"이 끊기지 않도록 하세요.

## 7. 2025 디자인 트렌드

### 🎨 시각 스타일 (Visual Style)

#### 플랫 2.0 (Flat 2.0)
- 완전한 평면이 아닌 **미세한 그림자(Drop Shadow)** 사용
- 깊이감과 계층 구조 표현
- 카드 UI에 2-4px 그림자로 떠있는 느낌

#### 다크 모드 (Dark Mode)
- 어두운 배경 + 밝은 타이포그래피
- 눈의 피로 감소, 특히 온라인 발표 시 효과적
- **네온/비비드 악센트 컬러**(초록, 파랑, 주황)와 조합

#### 글래스모피즘 (Glassmorphism)
- 카드나 오버레이에 **반투명 흐림 효과(Frosted Glass)** 적용
- 배경과 자연스럽게 어우러지면서 계층 구분

### 📝 타이포그래피 및 레이아웃

#### 액션 타이틀 (Action Titles)
- 분류형 제목 ❌: "매출 분석"
- **통찰 요약형 제목** ✅: "신제품 출시로 매출이 20% 성장했습니다"
- 제목만 읽어도 핵심 메시지 파악 가능

#### 벤토 그리드 (Bento Grid)
- Apple/IT 마케팅에서 유행하는 **모듈형 레이아웃**
- 다양한 크기의 박스에 기능, 지표, 이미지 배치
- 대시보드처럼 정보를 한눈에 파악

### 🎬 마이크로 애니메이션
- 페이드인, 프로그레스 바, 슬라이드 전환
- 과하지 않게 주의를 유도
- 스토리텔링 강화

### 📐 아이소메트릭 디자인
- 2.5D 효과로 깊이감 표현
- 기술 아키텍처, 워크플로우 시각화에 적합
- 레이어드 데이터 시스템 표현

### 🧹 미니멀리즘 강화
- 내용 제거가 아닌 "집중"
- 하나의 슬라이드 = 하나의 메시지
- 여백의 전략적 활용

### 📱 모바일 퍼스트 고려
- 작은 화면에서도 가독성 확보
- 큰 폰트, 단순한 차트
- 세로형 슬라이드 증가 추세
