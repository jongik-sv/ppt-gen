# 개념 충돌 해결 (Lite)

> 코드/설계서 생성 시 개념 충돌 해결 규칙

---

## 충돌 유형

| 유형 | 상황 | 해결 |
|------|------|------|
| **A** | 코드 vs 설계서 | 설계서 우선 |
| **B** | 설계서 vs 설계서 | 사용자 확인 |
| **C** | 설계서 vs 기존 코드베이스 | 설계서 우선 |

---

## 설계 문서 우선순위

```
1순위: 020-detail-design.md (상세설계)
2순위: 010-basic-design.md (기본설계)
3순위: 011-ui-design.md (화면설계)
```

---

## 해결 프로세스

**Type A, C**: 설계 문서 기준 수정 → 변경 이력 기록

**Type B**: AskUserQuestion으로 사용자 확인 → 선택에 따라 진행 → 미선택 문서 업데이트

---

