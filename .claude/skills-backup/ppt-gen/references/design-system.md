# Design System

PPT 자동 생성 시 따라야 할 디자인 원칙 및 가이드라인.

---

## 1. 슬라이드 크기

### 지원 비율

| 비율 | 크기 (px) | 크기 (pt) | 용도 |
|------|----------|----------|------|
| **16:9** | 720 × 405 | 540 × 304 | 프레젠테이션 (기본) |
| 4:3 | 720 × 540 | 540 × 405 | 레거시, 인쇄 |
| 16:10 | 720 × 450 | 540 × 338 | 노트북 화면 |

### 권장 사항

- 16:9 비율 기본 사용
- 모든 요소는 슬라이드 경계 안에 배치
- 안전 영역: 가장자리에서 30px 이상 여유

## 2. 타이포그래피

### 폰트 선택

**웹 안전 폰트만 사용** (PowerPoint 호환):
- Arial, Helvetica (산세리프)
- Times New Roman, Georgia (세리프)
- Courier New (고정폭)
- Verdana, Tahoma, Trebuchet MS

### 크기 계층

| 요소 | 크기 범위 | 권장 |
|------|----------|------|
| 메인 타이틀 | 36-72pt | 48pt |
| 부제목 | 24-36pt | 28pt |
| 섹션 제목 | 20-28pt | 24pt |
| 본문 | 14-18pt | 16pt |
| 캡션/각주 | 10-12pt | 11pt |
| 강조 숫자 | 48-72pt | 60pt |

### 대비 원칙

```
극단적 대비 권장:
- 타이틀: 48-72pt (크게)
- 본문: 11-14pt (작게)

효과: 시각적 계층 명확화, 주목도 향상
```

### 굵기 사용

| 용도 | 굵기 |
|------|------|
| 제목 | Bold (700) |
| 부제목 | Semi-bold (600) |
| 본문 | Regular (400) |
| 강조 | Bold |

## 3. 레이아웃

### 여백 (Margin)

```
외부 여백: 30px (슬라이드 가장자리)
내부 패딩: 40px (콘텐츠 박스 내부)
요소 간격: 20-30px
```

### 2열 레이아웃 (권장)

```
┌─────────────────────────────────────┐
│  제목 (전체 너비)                    │
├──────────────┬──────────────────────┤
│   텍스트     │    차트/이미지        │
│   (40%)      │      (60%)           │
└──────────────┴──────────────────────┘

권장 비율: 30/70, 40/60, 25/75
```

### 레이아웃 유형

| 유형 | 설명 | 용도 |
|------|------|------|
| 중앙 정렬 | 제목+부제목 중앙 | 표지, 섹션 구분 |
| 2열 비대칭 | 텍스트 + 비주얼 | 본문, 데이터 |
| 카드 그리드 | 3-4열 카드 | 통계, 기능 소개 |
| 타임라인 | 가로/세로 흐름 | 일정, 프로세스 |
| 비교 | 좌우 대칭 | A vs B, 장단점 |

### 금지 레이아웃

- 세로 스택 (콘텐츠 위-아래 배치) → 공간 낭비
- 3열 이상 (콘텐츠 부족 시) → 산만함
- 중앙 정렬 본문 → 가독성 저하

## 4. 정렬 규칙

### 텍스트 정렬

| 콘텐츠 유형 | 정렬 |
|------------|------|
| 제목 | 좌측 또는 중앙 |
| 본문/불릿 | 좌측 |
| 숫자/통계 | 중앙 또는 우측 |
| 캡션 | 좌측 |

### 요소 정렬

- 수평선 기준 정렬 유지
- 수직 그리드 일관성
- 시각적 균형 고려

## 5. 불릿 포인트

### 기본 규칙

```html
<!-- 올바른 사용 -->
<ul>
  <li>첫 번째 항목</li>
  <li>두 번째 항목</li>
  <li>세 번째 항목</li>
</ul>

<!-- 금지: 수동 기호 -->
<p>• 항목</p>   <!-- X -->
<p>- 항목</p>   <!-- X -->
<p>* 항목</p>   <!-- X -->
```

### 제한

| 항목 | 권장 |
|------|------|
| 최대 개수 | 4-5개 |
| 최대 깊이 | 2단계 |
| 한 줄 길이 | 2줄 이내 |

### 스타일

- 간결한 문장 (주어 생략 가능)
- 병렬 구조 유지
- 동일한 문법 패턴

## 6. 색상 사용

### 역할별 색상

| 역할 | 용도 |
|------|------|
| Primary | 제목, 강조, CTA |
| Secondary | 부제목, 보조 강조 |
| Text | 본문 텍스트 |
| Background | 슬라이드/박스 배경 |
| Accent | 하이라이트, 아이콘 |

### 대비 규칙

```
텍스트-배경 대비:
- 어두운 배경 → 밝은 텍스트 (흰색)
- 밝은 배경 → 어두운 텍스트 (진회색)

권장 명도차: 최소 4.5:1 (WCAG AA)
```

### 색상 개수

- 슬라이드당 3-5색
- 메인 1색 + 보조 1-2색 + 강조 1색

---

## 7. 차트 스타일

### 기본 원칙

- **단순화**: 불필요한 요소 제거
- **직접 레이블**: 범례 대신 데이터 포인트에 라벨
- **단색 기반**: 주요 데이터만 강조색

### 권장 스타일

```javascript
{
  showLegend: false,        // 범례 숨김
  showValue: true,          // 값 표시
  showCatAxisTitle: false,  // 축 제목 숨김
  showValAxisTitle: false,
  chartColors: ['4472C4'],  // 단일 색상
  dataBorder: { pt: 0 }     // 테두리 없음
}
```

### 차트 유형 선택

| 데이터 유형 | 권장 차트 |
|------------|----------|
| 비교 | 막대 (수평 권장) |
| 추세 | 라인 |
| 비율 | 도넛 (파이 대신) |
| 분포 | 산점도 |

### 색상 시퀀스

```javascript
// 다중 시리즈
chartColors: ['4472C4', 'ED7D31', 'A5A5A5', 'FFC000']

// 강조
chartColors: ['E0E0E0', 'E0E0E0', '4472C4', 'E0E0E0']
// 세 번째 항목만 강조
```

## 8. 이미지/아이콘

### 이미지 규칙

- 고해상도 사용 (최소 150 DPI)
- 비율 유지 (찌그러짐 방지)
- 적절한 크기 (슬라이드의 30-60%)

### 아이콘 규칙

- SVG → PNG 변환 필수 (html2pptx 제약)
- 최소 크기: 48×48px
- 권장 크기: 64×64px 이상
- 일관된 스타일 (선 굵기, 색상)

### 배경 이미지

- 어두운 오버레이로 텍스트 가독성 확보
- `rgba(0,0,0,0.4)` 권장

## 9. 시각적 효과

### 그림자

```css
/* 권장 */
box-shadow: 4px 4px 8px rgba(0,0,0,0.15);

/* 피해야 할 것 */
box-shadow: 10px 10px 20px rgba(0,0,0,0.5);  /* 너무 강함 */
```

### 둥근 모서리

```css
/* 권장 */
border-radius: 8px;   /* 버튼, 카드 */
border-radius: 50%;   /* 원형 아이콘 */

/* 피해야 할 것 */
border-radius: 30px;  /* 애매한 둥글기 */
```

### 테두리

```css
/* 권장 */
border: 1px solid #E0E0E0;  /* 미세한 구분 */
border-left: 4px solid #4472C4;  /* 강조 */

/* 피해야 할 것 */
border: 3px solid #000000;  /* 너무 두껍고 강함 */
```

## 11. 모범 사례

### DO (권장)

- 슬라이드당 핵심 메시지 1개
- 연속 3장 동일 레이아웃 피하기
- 비대칭 2열 레이아웃 활용
- 여백 충분히 확보
- 텍스트보다 비주얼 우선

### DON'T (금지)

- 한 슬라이드에 너무 많은 정보
- 작은 폰트 (12pt 미만)
- 복잡한 애니메이션 의존
- 저해상도 이미지
- 과도한 색상 사용 (5색 초과)
